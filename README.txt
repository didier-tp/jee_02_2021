la version complexe/complete jeeApp (packaging ear) necéssite une version complète de jboss wildfly (full profile avec EJB remote et MDB)
la version simple jeeWpApp (packaging war) peut se déployer sur une version allégée de jboss wildfly (web profile avec EJB local seulement)
Dans les 2 cas , des WS REST peuvent être codés/intégrés via JAX-RS
